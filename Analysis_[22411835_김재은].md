## 1. Introduction
 Price Tracking Secretary는 사용자가 상품의 링크와 자신이 구매하길 원하는 목표 가격을 등록하면 시스템이 모의 데이터를 생성해 해당 물품의 가격을 주기적으로 업로드하고 목표가 이하로 가격이 떨어질 때 알림을 보내주는 서비스이다. 이를 이용해 사용자는 같은 상품 페이지를 지속적으로 확인해야하는 불편함과 시간낭비를 줄일 수 있고 가격 변동 흐름 저장 기능 등을 통하여 적절한 구매 시점을 판단할 수 있도록 도와줄 수 있다. 
 이번 문서에서는 Price Tracking Secretary의 주요 기능과 구조를 정리하기 위해 사용자가 서비스를 이용하는 과정, 이 서비스에 필요한 주요 class, prototype 구현 및 간단 제시 등을 작성하였다.

### 3. domain analysis

1. User
    <br>: 가입한 사용자들의 정보를 Price Tracking Secretary에 관리하고 저장하는 클래스다.
    <br> ( ex : 아이디, 비밀번호, 이메일, 등록된 상품 목록)
 
2. Admin
    <br>: 관리자의 정보를 저장하는 클래스다.
    <br> ( ex : 관리자 아이디, 비밀번호, 접근 권한)

3. Product
    <br>: 사용자가 등록한 상품의 정보를 저장하는 클래스다.
    <br> ( ex : 상품명, 상품 링크, 현재 가격, 목표 가격, 상품 상태)

4. PriceHistory
    <br>: 등록된 상품의 가격의 변동 기록을 저장하는 클래스다.
    <br> ( ex : 상품 ID, 수집 시간, 가격)

5. Notification
    <br>: 사용자에게 상품 가격 정보에 대한 알림을 보낼 때 그 알림들의 내용을 저장하는 클래스다.
    <br> ( ex : 상품명, 현재 가격, 목표 가격, 알림 메세지, 알림 시간, 읽음 여부)

6. ErrorLog
    <br>: 가격 수집 실패나 알림 전송 실패 등 시스템에서 생기는 오류와 로그를 기록하는 클래스다.
    <br> ( ex : 오류 종류, 오류 메세지, 발생 시간, 관련 상품 정보)

7. Login
    <br>: 사용자가 시스템에 접속할 때 쓴 아이디, 비밀번호가 서버에 존재하는 정보인지 구분하는 기능을 가진 클래스다.
    <br> ( ex : 아이디 및 비밀번호 확인, 로그인 성공 여부 반환)

8. ProductRegister
    <br>: 시스템에 상품 사이트로 연결되는 상품 링크와 목표 가격을 저장하는 클래스다.
    <br> ( ex : 상품 링크 및 목표 가격 입력, 상품 정보 저장)

9. ProductManager
    <br>: 사용자가 등록한 상품을 조회 및 삭제하고, 목표가 변경 기능을 가진 클래스다.
    <br> ( ex : 관심 상품 목록 조회, 상품 삭제, 목표 가격 변경)

10. PriceTracker
    <br>: 정해진 시간마다 상품 가격을 확인하고 가격 이력을 저장하는 클래스다.
    <br> ( ex : 현재 가격 확인, 이력 저장, 상품 상태 갱신 )

11. Scheduler
    <br>: 정해진 시간마다 가격 추적 기능과 알림 전송 기능이 실행되게 하는 클래스다.
    <br> ( ex : 가격 수집 시간 확인, 가격 추적 실행, 알림 조건 확인 )

12. NotificationManager
    <br>: 목표 가격에 도달했는지 확인하고 사용자에게 알림을 전송하는 기능을 가진 클래스다.
    <br> ( ex : 목표 가격 비교, 중복 알림 확인, 웹 또는 이메일 알림 전송 )

| 관리자 | 시스템의 오류나 로그를 확인하고 서비스 상태를 점검하는 운영 주체 |
| 상품 | 사용자가 가격 추적 대상으로 등록하는 물건 |
| 상품 링크 | 사용자가 시스템에 입력하는 상품 페이지의 주소 |
| 목표 가격 | 사용자가 알림을 받고자 직접 설정한 기준 금액 |
| 현재 가격 | 시스템이 가장 최근에 확인한 상품의 가격 |
| 가격 이력 | 시간에 따라 저장된 상품 가격 기록 |
| 가격 추적 | 상품 가격을 일정 간격으로 확인하면서 변동 여부를 확인하는 기능 |
| 알림 | 현재 상품의 가격이 목표가보다 작거나 같아진 경우 사용자에게 전달되는 안내 메시지 |
| 관심 상품 | 사용자가 가격 변화를 확인하기 위해 등록해 둔 상품 |
| 가격 수집 | 외부 쇼핑몰에서 현재 상품 가격 정보를 가져오는 과정 |
| 수집 주기 | 시스템이 가격 정보를 다시 확인하는 시간 간격 |
| 외부 쇼핑몰 | 시스템 밖에서 실제 상품 정보와 가격을 제공하는 온라인 쇼핑 사이트 |
| 마이페이지 | 사용자가 등록한 상품을 모아서 확인 가능하며, 이외 알림 설정 등을 만질 수 있는 페이지 |
| 로그 | 시스템 실행 기록이나 오류 내용을 남겨 두는 데이터 |
| 시스템 | 사용자 정보, 상품 정보, 가격 기록, 알림 내역 등을 저장하고 전반적인 업무를 처리하는  |
| 예외 상황 | 품절, 링크 오류, 가격 수집 실패처럼 정보가 정상적으로 처리되지 않는 경우 |
| 웹 알림 | 웹페이지 안에서 바로 확인할 수 있는 알림 방식 |
| 이메일 알림 | 사용자의 이메일 주소로 전달되는 알림 방식 |
