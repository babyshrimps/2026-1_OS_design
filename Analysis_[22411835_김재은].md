### 1. Introduction
 Price Tracking Secretary는 사용자가 상품의 링크와 자신이 구매하길 원하는 목표 가격을 등록하면 시스템이 링크를 통해 해당 물품의 가격을 주기적으로 확인하고 목표가 이하로 가격이 떨어질 때 알림을 보내주는 서비스이다. 이를 이용해 사용자는 같은 상품 페이지를 지속적으로 확인해야하는 불편함과 시간낭비를 줄일 수 있고 가격 변동 흐름 저장 기능 등을 통하여 적절한 구매 시점을 판단할 수 있도록 도와줄 수 있다. 
 이번 문서에서는 Price Tracking Secretary의 주요 기능과 구조를 정리하기 위해 사용자가 서비스를 이용하는 과정, 이 서비스에 필요한 주요 class, prototype 구현 및 간단 제시 등을 작성하였다.

### 6. Glossary  
| Terms | Description |
|---|---|
| 사용자 | 상품을 등록하고 목표 가격을 설정하며, 가격 변동을 확인하고 알림을 받는 서비스 이용자 |
| 관리자 | 시스템의 오류나 로그를 확인하고 서비스 상태를 점검하는 운영 주체 |
| 상품 | 사용자가 가격 추적 대상으로 등록하는 물건 |
| 상품 링크 | 사용자가 시스템에 입력하는 상품 페이지의 주소 |
| 목표 가격 | 사용자가 알림을 받고자 직접 설정한 기준 금액 |
| 현재 가격 | 시스템이 가장 최근에 확인한 상품의 가격 |
| 가격 이력 | 시간에 따라 저장된 상품 가격 기록 |
| 가격 추적 | 상품 가격을 일정 간격으로 확인하면서 변동 여부를 확인하는 기능 |
| 알림 | 현재 상품의 가격이 목표가보다 작거나 같아진 경우 사용자에게 전달되는 안내 메시지 |
| 관심 상품 | 사용자가 가격 변화를 확인하기 위해 등록해 둔 상품 |
| 가격 수집 | 외부 쇼핑몰에서 현재 상품 가격 정보를 가져오는 과정 |
| 수집 주기 | 시스템이 가격 정보를 다시 확인하는 시간 간격 |
| 외부 쇼핑몰 | 시스템 밖에서 실제 상품 정보와 가격을 제공하는 온라인 쇼핑 사이트 |
| 마이페이지 | 사용자가 등록한 상품을 모아서 확인 가능하며, 이외 알림 설정 등을 만질 수 있는 페이지 |
| 로그 | 시스템 실행 기록이나 오류 내용을 남겨 두는 데이터 |
| 시스템 | 사용자 정보, 상품 정보, 가격 기록, 알림 내역 등을 저장하고 전반적인 업무를 처리하는  |
| 예외 상황 | 품절, 링크 오류, 가격 수집 실패처럼 정보가 정상적으로 처리되지 않는 경우 |
| 웹 알림 | 웹페이지 안에서 바로 확인할 수 있는 알림 방식 |
| 이메일 알림 | 사용자의 이메일 주소로 전달되는 알림 방식 |
